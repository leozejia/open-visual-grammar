# Treatments / 视觉处理法

Treatment 是表面处理系统。

它不是 canon。

Canon 先回答“这张图解决什么视觉问题”。Treatment 只在 canon 选定之后，调整画面的质感、媒介和完成度：

```text
canon -> score -> optional treatment -> runtime spec
范式 -> 谱面 -> 可选处理法 -> 执行规格
```

Treatment 可以决定：

- 纸张肌理还是玻璃质感；
- 像素完成度还是手绘线条；
- 低饱和水墨还是高对比海报；
- 手账拼贴还是几何简化。

## 当前迁移进来的处理法

| Treatment | 说明 |
| --- | --- |
| Eastern textured hand-drawing | 东方纸张肌理、松弛线条、低饱和，适合隐喻表达。 |
| Retro pixel | 8-bit / 16-bit 像素语言，适合极客文化和复古科技感。 |
| Narrative journal infographic | 手绘时间线、对比图、结构化手账信息图。 |
| Whimsical journal sketch | 水彩、马克笔、拼贴感，适合灵感和生活方式语境。 |
| Flowing minimal gaze | 复古极简、负空间、钢笔或蚀刻质感。 |
| Elegant simple hand-drawing | 精致线条、典雅配色、有温度的网页插画感。 |
| Minimal line sketch | 简约线稿、克制撞色、低饱和、大量留白。 |

## 规则

不要从 treatment 开始。

先选 canon，再判断是否需要 treatment。

如果某个 treatment 逐渐拥有自己的视觉问题、参考图、构图规则和反例，它才应该升级成 canon。
