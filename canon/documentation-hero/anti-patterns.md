# Documentation Hero Anti-Patterns

## Fake UI proof

Do not invent dashboards, buttons, file trees, or terminal output when the user
needs a real product state.

## Social-media heat

Huge warning text, red alarms, and public-argument posture make docs feel less
trustworthy.

## Empty abstraction

Soft gradients, floating cubes, and generic AI nodes do not teach the reader
anything about the operation.

## Too literal

If every object is a label from the article, the hero becomes a diagram without
the clarity of a real diagram.

Use metaphor, but keep the metaphor near enough for beginners.
