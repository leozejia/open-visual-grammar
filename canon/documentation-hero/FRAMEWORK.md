# Documentation Hero

Documentation Hero is a calm explanatory canon for public docs and tutorials.

It should make the document feel useful before the reader starts reading.

## Use when

- The target is public documentation, tutorial pages, or knowledge base entries.
- The reader may be a beginner.
- The page should feel trustworthy and long-lived.
- The visual needs to support the concept without shouting.

## Do not use when

- The goal is to win a feed click.
- A real screenshot is required for proof.
- The article depends on personal voice or public debate.
- The visual idea cannot be understood without the title.

## Visual problem

The reader should feel: "This page will help me understand and act."

The hero should translate an abstract operation into a legible system:

```text
context -> compression -> handoff
runtime -> model -> cost
document -> agent -> action
```

## Invariants

- Medium information density.
- Clear central object or system.
- Calm hierarchy.
- No fake product UI.
- No exaggerated crisis tone.
- Works without exact text inside the image.

## Composition model

Use one of these models:

- System map: a few connected objects with readable relationships.
- Workspace: documents, cards, terminals, or tools arranged with intent.
- Ledger: cost, cache, or state shown as a structured but non-literal surface.
- Handoff: one session, file, or agent passing state to the next.

## Information density

Medium.

Enough detail to reward inspection, but not enough to become a diagram the
reader must decode.

## Emotional temperature

Quiet-warm.

Documentation should feel competent, patient, and practical. It can have visual
personality, but it should not feel like an advertisement.

## Runtime compile slots

When compiling into an image prompt, define:

- target docs surface and exact aspect ratio;
- reader level;
- operational concept;
- central system metaphor;
- composition model;
- information density;
- emotional temperature;
- forbidden fake UI/text/logos.

When compiling into web or slide layout, reserve final text for deterministic
typesetting.

## Good fit

- Public docs hero images.
- Tutorial landing images.
- Conceptual article covers where trust matters more than conflict.

## Bad fit

- X covers that need strong first-second impact.
- Step-by-step UI instructions.
- Brand campaigns that need a human face or product photo.
