# Canon

A canon is a stable visual pattern.

It is not a style prompt. It is a reusable answer to a visual problem.

## Current canons

| Canon | Use when |
| --- | --- |
| `conflict-poster` | Public feed covers need a sharp first read and one real conflict. |
| `documentation-hero` | Public docs or tutorials need calm trust and operational clarity. |

## Admission rule

A new canon needs:

- repeatable structure;
- at least one reference image;
- explicit boundaries;
- anti-patterns;
- runtime compile slots.

If it only worked once, keep it as a finished example in the source project.
