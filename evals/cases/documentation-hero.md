# Case: Documentation Hero

## Job

Test whether a pattern can make a public documentation page feel trustworthy,
beginner-readable, and operational.

## Typical pattern

`patterns/documentation-hero/PATTERN.md`

## Scenario

A public docs page teaches a durable workflow:

```text
Put stable project context in files so agents can act without carrying every
detail in a long session.
```

## Required score fields

- Viewer
- Job
- Operational concept
- Central system metaphor
- Composition
- Density
- Temperature
- Forbidden

## Runtime target

Image generation for a docs hero or tutorial landing image.

## Three-card requirement

The three candidates should explore different system models:

- system map;
- workspace;
- handoff or ledger.

## Pass signals

- The page feels useful before reading.
- The image supports the concept without shouting.
- The visual could live for months without feeling like a trend graphic.
- It does not pretend to be exact UI.

## Fail signals

- Looks like a feed ad.
- Fake UI or unreadable product interface.
- Too much alarm or crisis tone.
- Too generic to explain the document.
