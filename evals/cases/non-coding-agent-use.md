# Case: Non-Coding Agent Use

## Job

Test whether the grammar can serve agent-tool adoption beyond programming.

## Typical pattern

Choose one pattern based on the channel:

- `conflict-poster` for public hooks;
- `documentation-hero` for public docs or tutorials.

## Scenario

A beginner wants to use an advanced agent tool for non-coding work:

```text
Research, documents, slides, operations, or personal knowledge work can be
handled by an agent if the task is framed concretely.
```

## Required score fields

- Viewer
- Job
- First read
- Agent action
- Human benefit
- Composition
- Density
- Temperature
- Forbidden

## Runtime target

Image generation for X cover, docs hero, or tutorial cover.

## Three-card requirement

The three candidates should avoid programming defaults and explore:

- documents or notes becoming action;
- messy inputs becoming a usable output;
- human operator directing an agent workspace.

## Pass signals

- The image does not assume the viewer is a developer.
- The agent feels practical, not magical.
- The task is concrete.
- The visual still fits the selected pattern.

## Fail signals

- Terminal/code imagery by default.
- Generic robot assistant.
- Overpromising full automation.
- Looks like a SaaS stock illustration.
