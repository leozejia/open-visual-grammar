# Example: Context Management Documentation Hero

This example shows how a visual score can guide image generation without storing
a one-off production prompt as the source of truth.

## Source problem

Readers need to understand that long agent sessions are powerful but require
context management. The public docs image should feel practical, not alarming.

## Selected canon

`canon/documentation-hero`

## Visual score

```text
Viewer: Beginner agent-tool user
Job: Make context management feel concrete and manageable
Canon: documentation-hero
First read: A long session becomes an organized handoff
Tension: Useful memory vs dirty state
Metaphor: Workspace with active context, archived handoff, and a light cost ledger
Composition: System map / workspace
Density: Medium
Temperature: Quiet-warm
Runtime: Image generation for a docs hero
Forbidden: fake product UI, random code, brand logos, alarm colors, unreadable labels
```

## Runtime

Image generation, then deterministic resize/crop by the consuming project.

## References

See:

## Critique

The useful part of the score is not the exact objects. It is the balance:

- the metaphor stays close enough for beginners;
- the density is medium, not poster-like;
- the emotional temperature stays documentation-safe;
- the runtime is not asked to generate exact UI proof.
